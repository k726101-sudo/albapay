# AlbaPay 급여 검증기 — 사용 매뉴얼

> **버전**: v2.0  
> **URL**: https://albapay-verify.web.app  
> **대상**: 파트너 기업, 내부 감사팀, 노무사  
> **최종 업데이트**: 2026-05-14

---

## 1. 개요

AlbaPay 급여 검증기는 AlbaPay 앱의 급여 계산 엔진을 독립적으로 검증할 수 있는 웹 도구입니다.

### 핵심 특징
- 🔒 **접근코드 기반 보안** — 승인된 파트너만 접속 가능
- 📊 **4가지 검증 모드** — 시급제, 월급제, 퇴직금, 상세검증
- 📋 **엑셀 붙여넣기** — 실제 출퇴근 데이터로 정밀 검증
- 🎛 **통상임금 구성 스위치** — 식대/고정OT/기타수당 개별 ON/OFF
- 📅 **스케줄 기반 자동 계산** — 출퇴근 기록 없이도 스케줄만으로 검증 가능
- 💾 **JSON 저장** — 검증 결과를 로컬 파일로 영구 보관
- 🚫 **서버 독립** — 별도 백엔드 서버 없이 운영 (Firebase Hosting + Firestore만 사용)

---

## 2. 접속 방법

### 접근코드 입력
1. https://albapay-verify.web.app 접속
2. 발급받은 **접근코드** 입력 (예: `TEST-001`)
3. `확인` 버튼 클릭 → 검증기 메인 화면 진입

### 접근코드 관리
- 관리자 페이지: `https://albapay-verify.web.app/#/admin`
- Firebase 콘솔에서 직접 관리: `Firestore → verify_access` 컬렉션

## 4. 탭별 상세 안내

### 4-1. 시급제 검증

**용도**: 시급 기반 근로자의 급여를 요약 입력으로 빠르게 검증

#### 입력 항목
| 구분 | 항목 | 설명 |
|------|------|------|
| **기본 정보** | 시급 | 근로자 계약 시급 (원) |
| | 주 소정근로시간 | 계약상 주간 근로시간 (예: 35h) |
| | 휴게시간 | 교대당 휴게시간 (분) |
| | 정산기간 | 시작일 ~ 종료일 |
| | 근무 요일 | 월~일 중 선택 |
| **근로시간 요약** | 총 순수근로시간 | 해당 정산기간 총 근로시간 (h) |
| | 만근 주 수 | 주 소정근로를 채운 주 수 |
| | 연장/야간/휴일 | 각각의 가산 근로시간 (h) |
| | 결근 주 수 | 주휴수당 미지급 주 |
| **옵션** | 5인 이상 | 5인 이상 사업장 여부 (가산수당 적용) |
| | 유급 휴게 | 휴게시간 유급 여부 |
| | 수습기간 | 수습 여부 (최저임금 90%) |

#### 계산 결과
- 기본급 = 순수근로시간 × 시급
- 가산수당 = (연장+야간+휴일) × 시급 × 0.5 (5인 이상만)
- 주휴수당 = (주 소정/40 × 8h) × 시급 × 만근주수
- 근로자의 날 수당 (정산기간에 5/1 포함 시)
- 4대보험 공제 → 실지급액

#### 예제 시나리오
| 코드 | 설명 |
|------|------|
| A-1 | 주 35시간 · 주5일 기본 시급제 |
| A-2 | 주 15시간 미만 · 주휴수당 비대상 |
| A-3 | 야간+연장 가산수당 포함 |

### 4-2. 월급제 검증

**용도**: 월급 기반 근로자의 급여 구조와 최저임금 적법성 검증

#### 입력 항목
| 구분 | 항목 | ON/OFF 스위치 | 설명 |
|------|------|:---:|------|
| **월급 구성** | 통상임금 / 기본급 | 항상 ON | 월 기본급 (예: 2,156,880원) |
| | 식대 | ✅ `식대 포함` | 월 식대 (예: 200,000원) |
| | 고정연장수당 | ✅ `고정OT 포함` | 포괄임금 고정OT (예: 140,000원) |
| | 고정연장시간 | (고정OT 연동) | 약정 연장근로시간 (h) |
| **기타 수당** | 기타(이외) 수당 | ✅ `포함` | 수당명 + 금액 직접 입력 |
| **일할 계산** | 입사일/퇴사일 | — | 중도 입퇴사 시 일할 계산 자동 |

> 💡 **통상임금 구성 스위치**: 각 항목의 ON/OFF 스위치로 통상임금에 포함할 구성요소를 자유롭게 설정할 수 있습니다. 노무사가 상황에 따라 식대·고정OT·기타수당을 개별적으로 켜고 끌 수 있어, 다양한 급여 구조를 빠르게 검증할 수 있습니다.

> 💡 **Placeholder 방식**: 금액 입력 필드는 비어 있으며, 투명한 예시 텍스트(예: `예) 2,156,880`)가 표시됩니다. 입력 시 자동으로 사라집니다.

#### 핵심 검증 포인트
- **최저임금 Hard Block**: 기본급 ÷ S_Legal < 법정 최저임금 → 🚨 미달
- **최저임금 Warning**: (기본급+식대) ÷ S_Ref < 최저임금 → ⚠️ 주의
- **통상시급**: 기본급 ÷ S_Legal (보수적 기준)
- **참고시급**: (기본급+식대) ÷ S_Ref (총보상 참고)
- **일할 계산**: 중도 입퇴사 시 일수 비례 자동 적용

#### 예제 시나리오
| 코드 | 설명 |
|------|------|
| B-1 | 250만원 표준 월급제 |
| B-2 | 중도 퇴사 일할 계산 |
| B-3 | 최저임금 미달 사례 |

### 4-3. 퇴직금 정산

**용도**: 퇴직금 + 잔여 연차수당 + 일할급여 통합 정산

#### 입력 항목
| 항목 | 설명 |
|------|------|
| 입사일 / 퇴사일 | 재직 기간 산정 |
| 시급 | 평균임금 계산 기준 |
| 주 소정근로시간 | 연차 및 퇴직금 산정 기준 |
| 사용 연차 | 이미 사용한 연차 일수 |
| 수동 연차 가감 | 사장님 수동 조정분 |
| 수동 평균임금 | 0이면 자동 계산, 직접 입력 가능 |
| 임금 유형 | 시급제 / 월급제 선택 |

#### 월급제 선택 시 — 통상임금 구성 스위치
| 항목 | ON/OFF 스위치 | 설명 |
|------|:---:|------|
| 통상임금 / 기본급 | 항상 ON | 월 기본급 (예: 2,156,880원) |
| 식대 | ✅ `포함` | 통상임금 포함 여부 선택 |
| 고정OT | ✅ `포함` | 통상임금 포함 여부 선택 |
| 기타 수당 | ✅ `포함` | 기타 수당 포함 여부 + 금액 입력 |

> 💡 스위치를 OFF하면 해당 금액이 **0원 처리**되며, 입력 필드가 **비활성(투명도 40%)** 상태로 변합니다.

#### 계산 결과
```
총 정산금액 = 잔여 연차수당 + 퇴직금 + 퇴사월 일할급여

퇴직금 = 1일 평균임금 × 30일 × (재직일수 / 365)
  └ 조건: 1년 이상 + 주 15시간 이상 근무

잔여 연차 = (발생 연차 - 사용 연차 ± 수동 가감) × 1일 통상임금

지급기한 = 퇴직일 + 14일
```

#### 예제 시나리오
| 코드 | 설명 |
|------|------|
| D-1 | 시급제 2년 근무 후 퇴직 |
| D-2 | 월급제 1년 근무 후 퇴직 |
| D-3 | 1년 미만 퇴직 (퇴직금 비대상) |

### 4-4. 상세검증 (출퇴근 데이터 기반)

**용도**: 실제 출퇴근 기록 또는 스케줄 기반으로 PayrollCalculator 전체 엔진을 실행

> ⚡ 가장 정밀한 검증 모드 — 주휴, 연장, 야간, 휴일 모두 실제 기록 기반 산정

#### 근무 스케줄 입력
스케줄 초과 근무시간이 **연장근무(가산수당)**로 산정됩니다.

| 항목 | 기본값 | 설명 |
|------|--------|------|
| 출근 시간 | 09:00 | 계약상 근무 시작 시간 |
| 퇴근 시간 | 18:00 | 계약상 근무 종료 시간 |

예) 스케줄 18:00, 실제 퇴근 20:00 → **2시간 연장근무**

#### 수동 근로 데이터 입력
출퇴근 기록 없이 **스케줄만으로 계산**할 때 사용하는 보정 항목:

| 항목 | 단위 | 설명 |
|------|------|------|
| 연장근로 | 시간 (h) | 총 연장근로 시간 → 근무일에 균등 분배 |
| 야간근로 | 시간 (h) | 총 야간근로 시간 → 앞쪽 근무일부터 22:00 이후 배분 |
| 결근 주수 | 주 | **만근 실패 주 수** → 해당 주에서 1일 제거 → 주휴수당 차감 |

> ⚠️ **결근 주수**: "결근 일수"가 아닌 "결근 주수"입니다. 입력한 숫자만큼의 주에서 만근이 실패하여 해당 주의 **주휴수당이 차감**됩니다. (예: 2 입력 → 마지막 2주의 주휴수당 미발생)

#### 출퇴근 데이터 입력 형식

**지원하는 구분자**: 탭(↹), 콤마(,), 공백 2칸 이상

```
날짜           출근    퇴근
2026-05-16    09:00   18:00
2026-05-19    09:00   20:00    ← 연장근무
2026-06-03    14:00   23:00    ← 야간근무
```

**지원하는 날짜 형식**:
| 형식 | 예시 |
|------|------|
| yyyy-MM-dd | 2026-05-16 |
| yyyy/MM/dd | 2026/05/16 |
| yyyy.MM.dd | 2026.05.16 |
| yyyyMMdd | 20260516 |

**지원하는 시간 형식**:
| 형식 | 예시 |
|------|------|
| HH:mm | 09:00, 18:00 |
| H:mm | 9:00 |
| HHmm | 0900 |

#### 스케줄 기반 자동 계산 (출퇴근 기록 미입력 시)
출퇴근 데이터를 입력하지 않으면, 설정한 **근무 스케줄**을 기준으로 자동으로 출퇴근 기록을 생성합니다:

1. 정산기간 내 근무 요일에 해당하는 날짜마다 출근/퇴근 기록 자동 생성
2. 결근 주수만큼 해당 주에서 1일 제거 (주휴수당 차감)
3. 연장근로 시간을 근무일에 균등 분배 (퇴근시간 연장)
4. 야간근로 시간을 앞쪽 근무일에 22:00 이후로 배분

#### 사용 방법
1. 엑셀/구글시트에서 날짜·출근·퇴근 3열을 선택 → 복사 (Ctrl+C)
2. 데이터 입력란에 붙여넣기 (Ctrl+V)
3. 자동 파싱 → 인식 결과 확인 (텍스트 변경 시 자동 트리거)
4. **"급여 계산"** 버튼 클릭 → 전체 엔진 실행

> 💡 출퇴근 데이터 미입력 시, 스케줄 기반 자동 계산이 실행됩니다.

#### 예제 버튼
| 버튼 | 내용 |
|------|------|
| 🧪 만근 예제 | 주5일 09:00~18:00 × 21일 정상 만근 |
| ⏱ 연장+야간 예제 | 연장(~20:00, ~21:00) + 야간(14:00~23:00) 포함 |

#### 참고사항
- 헤더 행(날짜, date 등)은 자동으로 건너뜀
- 퇴근 시간이 출근보다 이르면 **익일 퇴근**으로 자동 처리
- 최대 25건까지 미리보기 표시

## 5. 검증 결과 저장

### JSON 저장
모든 탭에서 계산 완료 후 **"검증 결과 JSON 저장"** 버튼을 클릭하면 다운로드 폴더에 파일이 저장됩니다.

#### 파일명 규칙
```
검증_{업체명}_{직원이름}_{검증유형}_{날짜}.json

예) 검증_노무법인ABC_김영수_시급제_2026-05-14.json
```

#### 저장 데이터 구조
```json
{
  "meta": {
    "firmName": "노무법인 ABC",
    "workerName": "김영수",
    "workerNumber": "EMP-001",
    "verifyType": "시급제",
    "createdAt": "2026-05-14T14:10:00.000",
    "appVersion": "AlbaPay Verify v2.0"
  },
  "input": {
    "hourlyWage": "10320",
    "weeklyHours": "35",
    "totalPureHours": "150.5",
    "..."
  },
  "result": {
    "basePay": 1553160,
    "weeklyHolidayPay": 289920,
    "totalPay": 1843080,
    "netPay": 1843080
  }
}
```

> 💡 상세검증 탭에서는 출퇴근 기록 원본(`attendanceData`)도 함께 저장됩니다.

## 7. 접근코드 관리 (관리자)

### Firestore 구조
```
verify_access/{코드ID}
├── firmName: "노무법인 ABC"    (string)
├── enabled: true               (boolean)
├── createdAt: timestamp
└── lastAccessedAt: timestamp   (자동 갱신)
```

### 새 코드 발급 방법
**방법 1**: Firebase 콘솔
1. Firestore → `verify_access` 컬렉션
2. 문서 추가 → ID: `FIRM-001`
3. 필드: `firmName` = "노무법인 ABC", `enabled` = `true`

**방법 2**: 관리자 페이지

## 11. FAQ

**Q: 계산 결과가 앱과 다를 수 있나요?**  
A: 시급제/월급제 탭은 요약 입력 기반 직접 계산으로, 앱 엔진과 약간의 차이가 있을 수 있습니다. **상세검증 탭**은 앱과 동일한 `PayrollCalculator.calculate()` 엔진을 사용하므로 결과가 일치합니다.

**Q: 월급제에서 식대/고정OT 스위치를 끄면 어떻게 되나요?**  
A: 해당 항목이 **0원 처리**되어 통상임금 산정에서 제외됩니다. 입력 필드도 비활성(투명도 40%)으로 변합니다. 노무사가 상황에 따라 통상임금 구성을 자유롭게 조절할 수 있습니다.

**Q: 결근 주수와 결근 일수의 차이는?**  
A: **결근 주수**는 만근하지 못한 주의 수입니다. 각 결근 주에서 1일이 제거되어 해당 주의 주휴수당이 차감됩니다. (예: 결근 주수 2 → 2주의 주휴수당 미발생)

**Q: 출퇴근 기록 없이도 검증 가능한가요?**  
A: 네. 상세검증 탭에서 출퇴근 데이터를 입력하지 않으면, 설정한 **근무 스케줄**을 기반으로 자동 계산됩니다. 연장/야간/결근은 수동 입력 필드로 보정할 수 있습니다.

**Q: 저장한 JSON 파일을 다시 불러올 수 있나요?**  
A: 현재는 파일 저장만 지원합니다. 향후 드래그&드롭 불러오기 기능이 추가될 예정입니다.

**Q: 접근코드를 분실했습니다.**  
A: 관리자에게 문의하여 새 접근코드를 발급받으세요. 기존 코드는 비활성화 처리됩니다.

**Q: 인터넷 없이 사용 가능한가요?**  
A: 최초 접속 시 접근코드 확인을 위해 인터넷이 필요합니다. 이후 계산 자체는 브라우저에서 로컬로 실행됩니다.

**Q: 배포 후 변경사항이 보이지 않습니다.**  
A: 브라우저 캐시 문제입니다. `Cmd + Shift + R` (강력 새로고침) 또는 시크릿 모드로 접속해보세요.
