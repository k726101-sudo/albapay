# 02. 출퇴근 판정 규칙 (Clock-In/Out Rules)

> 출처: `packages/shared_logic/lib/src/utils/roster_attendance.dart`

---

## 1. 출근 판정 흐름

### 1.1 출근 시각 보정 (`payrollEffectiveClockIn`)

```
입력: 실제 출근 시각(actualClockIn), 예정 출근 시각(scheduledStart), 유예시간(graceMinutes)

IF 실제 출근 ≤ 예정 출근 시각:
    → 급여 기준 출근 = 예정 출근 시각  (조기출근 → 예정 시각으로 보정)
    예: 08:50 도착, 09:00 예정 → 급여는 09:00부터 계산

ELSE IF 실제 출근 ≤ 예정 출근 + 유예시간:
    → 급여 기준 출근 = 예정 출근 시각  (유예 범위 내 지각 → 정시 출근 인정)
    예: 09:03 도착, 유예 5분 → 급여는 09:00부터 계산

ELSE:
    → 급여 기준 출근 = 실제 출근 시각  (유예 초과 지각 → 실제 시각 적용)
    예: 09:10 도착, 유예 5분 → 급여는 09:10부터 계산
```

> **법적 근거**: 근로기준법 제17조 (근로조건 명시), 대법원 판례 — 소정근로시간 이전 출근은 사용자 지시가 없는 한 근로시간에 포함하지 않음

### 1.2 자동 승인 판정 (`shouldAutoApproveClockIn`)

```
입력: 실제 출근 시각(now), 근무표 시프트(shift)

IF 실제 출근 ≤ 예정 출근 + 유예시간(기본 5분):
    → 자동 승인 (Normal)
ELSE:
    → 사장님 승인 대기 (pending_approval)
```

---

## 2. 퇴근 판정 흐름

### 2.1 퇴근 시각 보정 (`payrollSettlementClockOut`)

```
입력: 실제 퇴근 시각(actualClockOut), 예정 퇴근 시각(scheduledShiftEndIso), 연장승인(overtimeApproved)

IF 예정 퇴근 시각이 없음:
    → 급여 기준 퇴근 = 실제 퇴근 시각  (비예정 근무)

ELSE IF 연장근무 승인됨 OR 사장님 수정(isEditedByBoss):
    → 급여 기준 퇴근 = 실제 퇴근 시각  (초과 근무 전체 인정)

ELSE IF 실제 퇴근 > 예정 퇴근:
    → 급여 기준 퇴근 = 예정 퇴근 시각  (미승인 초과 → 예정 시각으로 절단)
    예: 18:20 퇴근, 18:00 예정, 미승인 → 급여는 18:00까지만 계산

ELSE:
    → 급여 기준 퇴근 = 실제 퇴근 시각  (조퇴 → 실제 퇴근 시각 적용)
```

> **핵심**: 연장근무가 승인되지 않으면, 예정 퇴근 이후 시간은 급여에 포함되지 않습니다.

### 2.2 조퇴 판정 (`earlyLeaveMinutes`)

```
입력: 실제 퇴근 시각(actualClockOut), 예정 퇴근 시각(scheduledEnd)

IF 실제 퇴근 < 예정 퇴근:
    → 조퇴 분수 = 예정 퇴근 - 실제 퇴근 (분 단위)
ELSE:
    → 조퇴 아님 (0분)
```

---

## 3. 지각 판정

### 3.1 지각 분수 계산 (`lateMinutes`)

```
입력: 실제 출근 시각(actualClockIn), 예정 출근 시각(scheduledStart)

IF 실제 출근 > 예정 출근:
    → 지각 분수 = 실제 출근 - 예정 출근 (분 단위)
ELSE:
    → 지각 아님 (0분)
```

---

## 4. 근무표 시프트 결정 (`effectiveShiftForDate`)

```
입력: 근로자 데이터(worker), 날짜(date), 개별 근무표(rosterDayDoc)

1단계: 개별 근무표(rosterDays/{YYYY-MM-DD})에 해당일 기록이 있는가?
   → YES: 개별 지정 시프트 사용 (우선순위 최고)
   → NO: 2단계로

2단계: 근로자의 기본 계약(workDays)에 해당 요일이 포함되는가?
   → YES: 기본 계약 시간(checkInTime/checkOutTime) 사용
   → NO: null 반환 (휴무일 → Unplanned 처리)
```

---

## 5. 유예시간(Grace Period) 적용 요약

| 상황 | 유예시간 기본값 | 적용 결과 |
|------|----------------|----------|
| 09:00 예정, 09:03 도착 | 5분 | **정시 인정** (09:00 기준 급여) |
| 09:00 예정, 09:06 도착 | 5분 | **지각** (09:06 기준 급여) |
| 09:00 예정, 08:50 도착 | - | **조기출근** (09:00 기준 급여) |
| 18:00 예정, 18:20 퇴근, 미승인 | - | **절단** (18:00 기준 급여) |
| 18:00 예정, 18:20 퇴근, 승인 | - | **인정** (18:20 기준 급여) |
| 18:00 예정, 17:30 퇴근 | - | **조퇴** (17:30 기준 급여) |

> **매장별 설정**: 유예시간은 `stores/{storeId}.attendanceGracePeriodMinutes`에서 매장별로 설정 가능합니다 (기본값: 5분).
