# 03. 급여 계산 핵심 엔진 (Payroll Engine Spec)

> 출처: `packages/shared_logic/lib/src/utils/payroll_calculator.dart`
> 핵심 메서드: `PayrollCalculator.calculate()`

---

## 1. 계산 엔진 개요

급여 계산은 **정산 기간(periodStart ~ periodEnd)** 내의 모든 출퇴근 기록을 순회하며 다음 항목을 산출합니다:

```
최종 급여 = 기본급 + 유급휴게수당 + 가산수당 + 주휴수당 + 기타수당 + 연차수당 - 공제
```

---

## 2. 시급제 계산 흐름

### 2.1 일별 순수 근로시간 산출

```
FOR EACH 출퇴근 기록 (attendance) IN 정산기간:
    
    // 1. 급여 포함 여부 판정
    IF status == 'rejected' → SKIP (급여 미반영)
    IF status == 'Unplanned' AND NOT isEditedByBoss → SKIP (미승인 비예정)
    IF clockOut == null → SKIP (퇴근 미기록)
    
    // 2. 퇴근 시각 보정
    effectiveOut = payrollSettlementClockOut(
        actualClockOut, scheduledShiftEndIso, overtimeApproved || isEditedByBoss
    )
    
    // 3. 체류 시간 = 퇴근 - 출근 (분)
    stayMinutes = effectiveOut - clockIn
    
    // 4. 휴게 시간 차감 (3단계 우선순위)
    breakMinutes = calculateAppliedBreak(...)
    
    // 5. 순수 근로시간
    pureMinutes = stayMinutes - breakMinutes
    
    // 누적
    totalPureLaborMinutes += pureMinutes
    totalStayMinutes += stayMinutes
```

### 2.2 기본급 계산

```
시급 = hourlyRate (매장 설정값)
기본급 = (totalPureLaborMinutes / 60) × 시급

법적 근거: 근로기준법 제2조 제1항 제5호 (임금)
```

### 2.3 유급 휴게 수당

```
IF isPaidBreak (매장 설정: 유급휴게 적용):
    유급휴게수당 = (totalPaidBreakMinutes / 60) × 시급
ELSE:
    유급휴게수당 = 0
```

## 3. 가산수당 (5인 이상 사업장에만 적용)

> **적용 조건**: `isFiveOrMore == true` (상시근로자 5인 이상)
> **법적 근거**: 근로기준법 제56조

### 3.1 연장근로 가산 (50%)

```
FOR EACH 출퇴근 기록:
    일별 순수근로시간 > 8시간인 경우:
        연장시간 = 순수근로시간 - 8시간 (480분)
        단, 고정연장시간(fixedOvertimeHours) 설정 시:
            초과분 = 연장시간 - 고정연장시간
            IF 초과분 > 0 → 추가 연장가산 발생

연장가산수당 = 연장시간 × 시급 × 0.5
```

### 3.2 야간근로 가산 (50%)

```
야간 시간대: 22:00 ~ 06:00

FOR EACH 출퇴근 기록:
    야간 체류시간 = overlap(근무시간, 22:00~06:00)
    야간 순수시간 = 야간 체류시간 × (순수근로시간 / 체류시간)  ← 비례 배분

야간가산수당 = 야간순수시간 × 시급 × 0.5

법적 근거: 근로기준법 제56조 제1항
```

### 3.3 휴일근로 가산 (50%)

```
휴일 판정:
  1. 근로자의 날(5/1) → 무조건 휴일
  2. 주휴일(weeklyHolidayDay) → 주 7일 근무자의 지정 휴일
  3. 계약 근무일이 아닌 날 → 비예정 출근(대타)

IF 휴일 출근:
    휴일가산수당 = 순수근로시간 × 시급 × 0.5

법적 근거: 근로기준법 제56조 제2항
```

## 4. 주휴수당

### 4.1 주휴수당 발생 조건

```
조건 1: 주 소정근로시간 ≥ 15시간
조건 2: 해당 주 만근 (= 계약 근무일 모두 출근)

법적 근거: 근로기준법 제55조 (유급주휴일)
```

### 4.2 주휴수당 계산 방식 (엔진 A: 실적 기반)

```
FOR EACH 표준주(월~일) IN 정산기간:
    // 해당 주의 출퇴근 기록 필터
    weeklyAttendances = 해당 주 출퇴근 기록
    
    // 주 순수 근로시간 합산
    weeklyPureHours = SUM(각 일별 순수근로시간)
    
    IF weeklyPureHours < 15.0 → 주휴수당 없음
    
    // 만근 판정 (엔진 B)
    IF 계약 근무일 모두 출근 (또는 연차/유급휴일로 간주):
        주휴시간 = weeklyPureHours / 40 × 8  (비례 배분)
        단, 주휴시간 ≤ 8시간 (상한)
        
        주휴수당 = 주휴시간 × 시급
```

### 4.3 만근 판정 규칙

```
만근 = 계약 근무일 전체에 대해:
  - clockOut이 존재하는 출퇴근 기록이 있거나
  - isAttendanceEquivalent == true (연차 사용일, 근로자의 날 등)

연차 사용일은 출근한 것으로 봄 (근로기준법 제60조 제6항)
```

## 5. 월급제 계산 흐름

### 5.1 월급제 구성 요소

```
총 월급 = 기본급 + 식대 + 고정연장수당

구성 요소 분해:
  S_Legal = 해당 월 실제 소정근로시간 (달력 기반 주수 계산)
  S_Ref   = 4.345 × (주 소정근로시간 + 주휴시간)  ← 고정 209h 기준

  통상시급 = 기본급 / S_Legal  ← 법정 수당 산출 기준 (보수적)
  참고시급 = (기본급 + 식대 + 기타) / S_Ref  ← 총보상 참고용
```

### 5.2 이중 안전장치 (S_Legal vs S_Ref)

```
최저임금 판정 (Hard Block):
  IF 기본급 / S_Ref < 최저임금(10,320원):
      → 경고: "기본급이 최저임금에 미달합니다"
      → 월급 구성 자체를 수정해야 함

최저임금 판정 (Warning):
  IF (기본급 + 식대) / S_Ref < 최저임금:
      → 주의: "식대 포함해도 최저임금에 미달합니다"

법적 근거: 최저임금법 제6조
```

### 5.3 일할 계산 (중도 입사/퇴사)

```
IF 정산기간 내에 입사일 또는 퇴사일이 있음:
    재직일수 = 정산기간 내 실제 재직 일수
    월전체일수 = 해당 월 총 일수
    
    proRataRatio = 재직일수 / 월전체일수
    
    기본급 = 계약 기본급 × proRataRatio
    식대   = 계약 식대 × proRataRatio
    고정OT = 계약 고정OT × proRataRatio
```

## 6. 근로자의 날 (5월 1일) 처리

```
근로자의 날은 유급휴일 (근로자의 날 제정에 관한 법률)

CASE 1: 출근하지 않은 경우
    → 유급 보장: 1일 소정근로시간 × 시급 (기본급에 포함)

CASE 2: 출근한 경우 (5인 이상)
    → 기본 유급 + 실제 근무시간 × 시급 × 1.5배 (통상임금 + 휴일가산)

CASE 3: 출근한 경우 (5인 미만)
    → 기본 유급 + 실제 근무시간 × 시급 × 1.0배 (가산수당 없음)
```

## 7. 공제 항목

### 7.1 4대 보험 (선택 적용)

```
과세 대상액 = 총급여 - 비과세 식대(월 20만원 한도)

국민연금     = 과세 대상액 × 4.5%
건강보험     = 과세 대상액 × 3.545%
장기요양보험 = 건강보험 × 12.95%
고용보험     = 과세 대상액 × 0.9%

법적 근거: 각 보험법 시행령
```

### 7.2 3.3% 원천징수 (프리랜서)

```
IF applyWithholding33:
    사업소득세 = 과세 대상액 × 3.0%
    지방소득세 = 사업소득세 × 10% (= 과세 대상액 × 0.3%)
```

### 7.3 최종 실지급액

```
실지급액 = 총급여 + 비과세식대 - 총공제액 + 전월정산금
```

## 9. 상시근로자 5인 이상/미만 전환(변동) 처리 로직

사업장의 규모 변화(채용, 퇴사 등)로 인해 상시근로자 수가 5인 이상에서 5인 미만으로, 혹은 그 반대로 변동될 경우의 처리 로직입니다.

### 9.1 급여 정산 기간(Payroll Period) 단위의 독립적 상태 고정 (Snapshot)

```
알바페이 시스템은 상시근로자 5인 여부(isFiveOrMore)를 매장 전체의 실시간 설정뿐만 아니라,
'각 월별 급여 정산 데이터(Payroll Summary)' 내부에 독립적인 스냅샷으로 영구 저장합니다.

- 정산 확정 시점의 상태 고정:
  사장님이 매월 급여를 계산/확정하는 시점에 "이번 정산 기간 동안 상시근로자 5인 이상이었습니까?" 
  라는 확인을 거치며, 해당 월의 급여 명세서에는 확정된 `isFiveOrMore` 값이 기록됩니다.

- 과거 급여의 무결성 보장:
  현재 매장이 5인 미만으로 떨어졌더라도, 과거 5인 이상 시절에 확정된 급여 명세서(가산수당 1.5배 지급 내역)는 
  현재 설정의 영향을 받지 않고 그대로 보존됩니다.
```

### 9.2 수당 및 연차 처리의 동적 변환

```
CASE 1: 5인 이상 → 5인 미만으로 전환 시
  - 가산수당 (연장/휴일/야간): 전환된 시점 이후의 정산 월부터는 가산수당 배율이 1.5배에서 1.0배(기본시급)로 자동 하향 적용됩니다.
  - 연차(Annual Leave): 5인 미만으로 전환된 달부터는 월별 만근 연차가 발생하지 않으며, 정기 연차 출근율 산정 대상에서도 제외됩니다. 단, 5인 이상 시절에 이미 발생하여 누적된 잔여 연차는 법적으로 소멸하지 않으며 그대로 사용하거나 수당으로 정산해야 합니다.

CASE 2: 5인 미만 → 5인 이상으로 전환 시
  - 가산수당: 전환된 정산 월부터 연장/야간/휴일 근무 시 1.5배의 가산수당이 자동 적용됩니다.
  - 연차: 전환된 시점부터 1년 미만자의 경우 1개월 만근 시 1개의 연차가 새롭게 발생하기 시작합니다.

법적 근거: 상시근로자 수 산정 기준 (근로기준법 시행령 제7조의2)
```

### 9.3 일시적 대타 근무로 인한 5인 이상 초과 시 방어 로직 (Edge Case)

```
상황 (Scenario): 
평소 5인 미만 사업장인데, 정산기간(1일~31일) 중 대타 근무자가 일시적으로 투입되어 특정 며칠만 하루 출근 인원이 5명을 넘긴 경우.

노무적 쟁점: 
하루라도 5인을 넘겼다고 해서 해당 정산 월 전체를 5인 이상 사업장으로 간주하고, 모든 직원에게 연장/야간 1.5배 가산수당을 줘야 하는가?

시스템 자동 방어 로직: 
1. 알바페이 엔진은 단순히 그 달의 최대 인원을 보지 않고 '1/2 예외 규정'을 정밀 추적합니다.
2. 정산기간 31일 중 대타가 투입되어 5인 이상이 된 날이 단 며칠(예: 3일)에 불과하다면, 나머지 28일은 5인 미만으로 가동된 것입니다.
3. 5인 미만 가동일수(28일)가 전체 영업일수(31일)의 절반(1/2)을 초과하므로, 시스템은 해당 월 전체를 최종적으로 '5인 미만 사업장'으로 확정(Lock)합니다.
4. 결과: 사장님은 일시적인 대타 기용 현상 때문에 예기치 않게 1.5배 가산수당 폭탄을 맞는 법적 위험(Legal Risk)에서 완벽하게 보호받게 됩니다.
```

### 9.4 특정 월 한정 5인 이상 전환 (월 단위 독립 판단)

```
상황 (Scenario): 
대타 등 여러 이유로 1/2 규정을 초과하여, 한 달(예: 30일) 중 절반 이상인 16일 동안 5인 이상이 근무한 경우.

노무적 쟁점: 
이 달에 5인 이상 기준을 넘겼다고 해서, 앞으로 매장 자체가 영구적으로 5인 이상 사업장으로 굳어지는 것인가?

시스템 자동 방어 로직: 
1. 노동법상 상시근로자 수는 '사유 발생일 전 1개월(산정기간)'을 단위로 매번 끊어서 새롭게 계산합니다.
2. 알바페이 엔진은 이를 반영하여 '1/2 규정'을 넘긴 해당 정산 월(1개월)에만 한정하여 '5인 이상' 상태를 스냅샷으로 적용합니다.
3. 따라서 10월에 16일을 넘겨 법적으로 5인 이상이 되어 1.5배 가산수당을 지급했더라도, 다음 달인 11월에 5인 이상 근무일이 다시 절반 미만(예: 10일)으로 떨어지면 시스템은 즉각 11월을 '5인 미만 사업장'으로 원상복구시킵니다.
```

### 9.5 실시간 상태 전환 지점 및 앱 내 경고 시스템 (위험 순간)

```
전환 지점(위험 순간)의 정확한 정의:
알바생이 늘어나서 법적 상태가 '5인 미만'에서 '5인 이상'으로 뒤집히는 위험한 순간은, 
수학적으로 '5인 이상 출근한 날'이 그 달의 가동일수 중 절반(1/2)에 도달하거나 초과하게 되는 바로 그날의 출근 시점입니다.

예시 (30일 영업 기준):
- [안전 지대]: 14일 동안 대타를 불러 하루 5인 이상이 근무했어도 절반(15일)을 넘지 않았으므로, 14일째까지는 아무리 직원이 많아 보여도 시스템은 법적으로 '5인 미만 사업장' 방어막을 유지합니다.
- [위험 순간]: 하루 더 대타가 출근하여 5인 이상 근무한 날이 15일째(딱 절반)가 되는 바로 그 출근 순간, 평균 인원에 관계없이 해당 월 전체가 법적으로 '5인 이상 사업장'으로 확정 전환됩니다.

앱 내 실시간 경고 시스템 (사후 알림):
```
