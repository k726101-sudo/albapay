1. 개요

AlbaPay 급여 검증기는 AlbaPay 앱의 급여 계산 엔진을 독립적으로 검증할 수 있는 웹 도구입니다.

핵심 특징

  • 🔒 접근코드 기반 보안 — 승인된 파트너만 접속 가능
  • 📊 4가지 검증 모드 — 시급제, 월급제, 퇴직금, 상세검증
  • 📋 엑셀 붙여넣기 — 실제 출퇴근 데이터로 정밀 검증
  • 💾 JSON 저장 — 검증 결과를 로컬 파일로 영구 보관
  • 🚫 서버 독립 — 별도 백엔드 서버 없이 운영

2. 접속 방법

접근코드 입력

  1. https://albapay-verify.web.app 접속
  2. 발급받은 접근코드 입력 (예: TEST-001)
  3. 확인 버튼 클릭 → 검증기 메인 화면 진입

4. 탭별 상세 안내

4-1. 시급제 검증

용도: 시급 기반 근로자의 급여를 요약 입력으로 빠르게 검증

입력 항목

구분항목설명
기본 정보시급근로자 계약 시급 (원)
주 소정근로시간계약상 주간 근로시간 (예: 35h)
휴게시간교대당 휴게시간 (분)
정산기간시작일 ~ 종료일
근무 요일월~일 중 선택
근로시간 요약총 순수근로시간해당 정산기간 총 근로시간 (h)
만근 주 수주 소정근로를 채운 주 수
연장/야간/휴일각각의 가산 근로시간 (h)
결근 주 수주휴수당 미지급 주
옵션5인 이상가산수당 적용 여부
유급 휴게휴게시간 유급 여부
수습기간최저임금 90% 적용

계산 공식

  • 기본급 = 순수근로시간 × 시급
  • 가산수당 = (연장+야간+휴일) × 시급 × 0.5 (5인 이상)
  • 주휴수당 = (주 소정/40 × 8h) × 시급 × 만근주수
  • 근로자의 날 수당 (정산기간에 5/1 포함 시)
  • 4대보험 공제 → 실지급액

예제 시나리오

코드설명
A-1주 35시간 · 주5일 기본 시급제
A-2주 15시간 미만 · 주휴수당 비대상
A-3야간+연장 가산수당 포함

4-2. 월급제 검증

용도: 월급 기반 근로자의 급여 구조와 최저임금 적법성 검증

입력 항목

구분항목설명
월급 구성기본급월 기본급 (원)
식대월 식대 (원)
고정연장수당포괄임금 고정OT (원)
고정연장시간약정 연장근로시간 (h)
추가 근로추가 연장고정OT 초과 연장시간
야간/휴일추가 야간·휴일 근로시간
일할 계산입사일/퇴사일중도 입퇴사 시 자동 일할

핵심 검증 포인트

  • 최저임금 Hard Block: 기본급 ÷ S_Legal < 법정 최저임금 → 🚨 미달
  • 최저임금 Warning: (기본급+식대) ÷ S_Ref < 최저임금 → ⚠️ 주의
  • 일할 계산: 중도 입퇴사 시 일수 비례 자동 적용

예제 시나리오

코드설명
B-1250만원 표준 월급제
B-2중도 퇴사 일할 계산
B-3최저임금 미달 사례

4-3. 퇴직금 정산

용도: 퇴직금 + 잔여 연차수당 + 일할급여 통합 정산

입력 항목

항목설명
입사일 / 퇴사일재직 기간 산정
시급평균임금 계산 기준
주 소정근로시간연차 및 퇴직금 산정 기준
사용 연차이미 사용한 연차 일수
수동 평균임금0이면 자동 계산, 직접 입력 가능
임금 유형시급제 / 월급제 선택

계산 공식

총 정산금액 = 잔여 연차수당 + 퇴직금 + 퇴사월 일할급여

퇴직금 = 1일 평균임금 × 30일 × (재직일수 / 365)
  └ 조건: 1년 이상 + 주 15시간 이상 근무

잔여 연차 = (발생 연차 - 사용 연차 ± 수동 가감) × 1일 통상임금

지급기한 = 퇴직일 + 14일

예제 시나리오

코드설명
D-1시급제 2년 근무 후 퇴직
D-2월급제 1년 근무 후 퇴직
D-31년 미만 퇴직 (퇴직금 비대상)

4-4. 상세검증 (출퇴근 데이터 기반)

용도: 실제 출퇴근 기록을 입력하여 PayrollCalculator 전체 엔진을 실행

⚡ 가장 정밀한 검증 모드 — 주휴, 연장, 야간, 휴일 모두 실제 기록 기반 산정

사용 방법

  1. 엑셀/구글시트에서 날짜·출근·퇴근 3열 선택 → 복사 (Ctrl+C)
  2. 데이터 입력란에 붙여넣기 (Ctrl+V)
  3. "데이터 파싱" 클릭 → 인식 결과 확인
  4. "급여 계산" 클릭 → 전체 엔진 실행

예제 버튼

버튼내용
🧪 만근 예제주5일 09:00~18:00 × 21일 정상 만근
⏱ 연장+야간 예제연장(~20:00, ~21:00) + 야간(14:00~23:00) 포함

5. 검증 결과 저장

JSON 저장

모든 탭에서 계산 완료 후 "검증 결과 JSON 저장" 버튼을 클릭하면 다운로드 폴더에 파일이 저장됩니다.

파일명 규칙

검증_{업체명}_{직원이름}_{검증유형}_{날짜}.json

예) 검증_ABC기업_김영수_시급제_2026-05-14.json

저장 데이터 구조

{
  "meta": {
    "firmName": "ABC 기업",
    "workerName": "김영수",
    "workerNumber": "EMP-001",
    "verifyType": "시급제",
    "createdAt": "2026-05-14T14:10:00.000",
    "appVersion": "AlbaPay Verify v1.0"
  },
  "input": { "hourlyWage": "10320", ... },
  "result": {
    "basePay": 1553160,
    "weeklyHolidayPay": 289920,
    "totalPay": 1843080,
    "netPay": 1843080
  }
}
💡 상세검증 탭에서는 출퇴근 기록 원본(attendanceData)도 함께 저장됩니다.

8. FAQ

Q: 계산 결과가 앱과 다를 수 있나요?
A: 시급제/월급제 탭은 요약 입력 기반 계산으로 약간의 차이가 있을 수 있습니다. 상세검증 탭은 앱과 동일한 엔진을 사용하므로 결과가 일치합니다.

Q: 저장한 JSON 파일을 다시 불러올 수 있나요?
A: 현재는 파일 저장만 지원합니다. 향후 드래그&드롭 불러오기 기능이 추가될 예정입니다.

Q: 접근코드를 분실했습니다.
A: 관리자에게 문의하여 새 접근코드를 발급받으세요.

Q: 인터넷 없이 사용 가능한가요?
A: 최초 접속 시 접근코드 확인을 위해 인터넷이 필요합니다. 이후 계산은 브라우저에서 로컬로 실행됩니다.
