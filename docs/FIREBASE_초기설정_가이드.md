# Firebase 초기 설정 – 단계별 가이드 (초보자용)

이 프로젝트는 **Firebase**를 쓰기 때문에, 앱/웹이 제대로 동작하려면 **Firebase 콘솔에서 몇 가지를 켜고 설정**해 줘야 합니다.  
아래 순서대로만 따라 하시면 됩니다.

---

## 준비

- **Firebase 콘솔 주소**: https://console.firebase.google.com/
- **Google 계정**으로 로그인한 뒤, 프로젝트 **`standard-albapay`** 를 선택합니다.  
  (목록에 없으면 “프로젝트 추가”로 먼저 만들거나, 이미 만든 프로젝트를 선택하세요.)

---

## 1단계: Authentication(로그인) 켜기

앱에서 “이메일/구글/애플/전화번호로 로그인”을 쓰려면 여기서 **각각 사용 설정**을 해 줘야 합니다.

1. 왼쪽 메뉴에서 **“빌드(Build)”** → **“Authentication”** 클릭
2. **“시작하기”** 또는 **“로그인 방법”** 탭 클릭
3. **“로그인 방법(Sign-in method)”** 탭에서 아래 항목들을 **사용 설정(Enabled)** 으로 바꿉니다.

| 로그인 방식 | 하는 일 | 설정 방법 |
|------------|--------|----------|
| **이메일/비밀번호** | 사장님 이메일 링크 로그인에 필요 | 해당 행 클릭 → “사용 설정” 켜기 → **“이메일 링크(비밀번호 없이 로그인)”** 도 켜기 → 저장 |
| **전화** | 알바가 전화번호(SMS)로 로그인 | 해당 행 클릭 → “사용 설정” 켜기 → 저장 |
| **Google** | 사장님 앱에서 구글 로그인(Android) | 해당 행 클릭 → “사용 설정” 켜기 → 저장 |
| **Apple** | 사장님 앱에서 애플 로그인(iOS) | 해당 행 클릭 → “사용 설정” 켜기 → 저장 |

- **전화** 로그인을 켜면 “SMS 할당량” 경고가 뜰 수 있습니다.  
  새 프로젝트는 하루 10건 등 제한이 있을 수 있으니, 테스트 후 필요하면 결제 계정 연결로 할당량을 늘릴 수 있습니다.

---

## 2단계: 승인된 도메인(Authorized domains) 확인

“어떤 웹 주소에서 로그인을 허용할지”를 정해 두는 곳입니다.  
알바 웹(Flutter Web)이나 이메일 링크 로그인이 동작하려면 **사용하는 도메인**이 여기 있어야 합니다.

1. **Authentication** 화면에서 **위쪽 톱니바퀴(설정)** 클릭
2. **“승인된 도메인(Authorized domains)”** 탭 클릭
3. 아래가 **반드시** 있는지 확인하고, 없으면 **“도메인 추가”** 로 넣어 줍니다.

- `localhost` (로컬 테스트용)
- `standard-albapay.firebaseapp.com`
- `standard-albapay.web.app`
- 알바 웹을 **다른 주소**에 배포했다면 그 주소도 추가 (예: `www.내도메인.com`)

---

## 3단계: Firestore Database 만들기

앱이 “매장/직원/출퇴근/공지” 같은 **데이터를 저장**하려면 Firestore를 켜야 합니다.  
한 번만 만들면 됩니다.

1. 왼쪽 메뉴 **“빌드”** → **“Firestore Database”** 클릭
2. **“데이터베이스 만들기”** 버튼 클릭
3. **위치** 선택  
   - 가까운 지역 선택 (예: `asia-northeast3` 서울) → “다음”
4. **보안 규칙** 선택  
   - 처음에는 **“테스트 모드에서 시작”** 을 선택해도 됩니다.  
   - (나중에 “규칙” 탭에서 “로그인한 사용자만” 등으로 꼭 바꿔 주세요.)
5. **“사용 설정”** 클릭

이후 콘솔에 “Cloud Firestore API를 사용 설정하세요” 같은 문구가 뜨면, 안내된 링크에서 **API 사용 설정**을 한 번 눌러 주면 됩니다.

---

## 4단계: 프로젝트에 앱이 등록되어 있는지 확인

이 프로젝트 코드에는 이미 **세 개의 앱**이 연결되어 있습니다.  
Firebase 콘솔에서 같은 것이 보이는지만 확인하면 됩니다.

1. 왼쪽 메뉴 **“프로젝트 개요”** 옆 **톱니바퀴** → **“프로젝트 설정”** 클릭
2. 아래로 내려서 **“내 앱”** 목록 확인

다음이 있으면 OK입니다.

| 앱 종류 | 패키지/식별자 | 용도 |
|--------|----------------|------|
| Android | `com.standard.albapay` | 사장님용 앱(Android) |
| iOS | `com.standard.albapay` | 사장님용 앱(iOS) |
| 웹 | 닉네임 아무거나, App ID에 `web` 포함 | 알바용 Flutter Web |

- **Android**: `google-services.json` 파일이 `apps/boss_mobile/android/app/` 에 있어야 합니다. (이미 있으면 설정 완료)
- **iOS**: `GoogleService-Info.plist` 가 `apps/boss_mobile/ios/Runner/` 에 있어야 합니다. (이미 있으면 설정 완료)
- **웹**: “Firebase SDK 스니펫” 또는 “구성”에 `authDomain: standard-albapay.firebaseapp.com` 같은 값이 보이면 됩니다.

---

## 5단계: (선택) 이메일 링크 로그인용 도메인

사장님이 **“이메일로 받은 링크”** 로 로그인하는 방식을 쓸 때,  
그 링크의 주소(도메인)도 “승인된 도메인”에 들어 있어야 할 수 있습니다.

- 링크 주소가 `https://albapay.page.link/...` 형태라면  
  **Authentication → 설정 → 승인된 도메인** 에 **`albapay.page.link`** 를 추가해 주세요.
- 다른 도메인을 쓴다면, 그 도메인을 추가하면 됩니다.

---

## 체크리스트 (한 번에 확인)

설정이 끝났다면 아래처럼 되어 있는지 확인해 보세요.

- [ ] Authentication → 로그인 방법에서 **이메일/비밀번호**(+ 이메일 링크), **전화**, **Google**, **Apple** 사용 설정됨
- [ ] Authentication → 설정 → **승인된 도메인**에 `localhost`, `standard-albapay.firebaseapp.com`, `standard-albapay.web.app` (그리고 실제 배포 도메인) 있음
- [ ] **Firestore Database** 가 생성되어 있고, 테스트 모드 또는 본인 규칙으로 “사용 설정” 상태
- [ ] 프로젝트 설정 → 내 앱에 **Android**, **iOS**, **웹** 앱이 등록되어 있음

---

## 문제가 생겼을 때

- **“권한 거부” / “PERMISSION_DENIED”**  
  → Firestore “규칙” 탭에서 규칙이 너무 엄격하지 않은지 확인. (테스트 모드였다가 나중에 “로그인한 사용자만” 등으로 변경)
- **“이 작업을 수행할 수 없습니다”**  
  → 해당 서비스(Authentication, Firestore)가 “사용 설정” 상태인지, API가 활성화되었는지 콘솔에서 확인
- **전화 로그인(SMS) 실패**  
  → Authentication → 로그인 방법에서 “전화”가 사용 설정인지, 승인된 도메인에 현재 접속한 웹 주소가 있는지 확인
