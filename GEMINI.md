# 프로젝트 룰 (AI 행동 규칙)

## 절대 규칙

1. **코드를 먼저 확인한 후 답변할 것** — 추측으로 "수정 필요", "버그 있음" 등의 판단을 내리지 않는다. 반드시 관련 코드를 열어보고 확인한 후에 답변한다. 코드 기반으로 대답하고 절대 추측해서 말하지 않는다.

2. **코드 수정은 사장님 승인 후에만 진행** — 어떤 수정이든 먼저 "이렇게 수정해도 될까요?"라고 확인을 받은 뒤 실행한다. 절대 마음대로 수정하지 않는다.

3. **한글로 대화할 것** — 모든 설명, 주석, 커밋 메시지 등은 한국어로 작성한다.

4. **문서 수정 시 데이터 정합성 확인** — 예시 데이터, 계산식, 시나리오를 수정할 때는 전체 맥락(근로자 유형, 근무시간, 급여 체계)이 논리적으로 일관되는지 반드시 확인한다.

5. **영향 범위를 먼저 분석할 것** — 코드 변경 전, 해당 함수/변수가 어디서 사용되는지 grep으로 전수 조사하고 영향 범위를 사장님에게 보고한 후 진행한다.
